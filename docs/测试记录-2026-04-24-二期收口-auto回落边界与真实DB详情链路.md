# 测试记录（2026-04-24）二期收口：auto 回落边界与真实 DB 详情链路

## investigate 结论
- 根因 1：`canReadFromDatabase()` 仅判断“库可达+表存在”，空 DB 也会被判定为 DB 模式，导致详情页与候选提交在 `auto` 下出现 404/失败。
- 根因 2：之前通过 `getPromptDetail`、`createPromptSubmission` 方法级 DB->fixture 回落规避 404，会造成来源口径分裂（列表可能来自 DB，详情/投稿来自 fixture）。
- 收口策略：把“是否走 DB”收敛到单点判定，`auto` 仅在 DB 具备最小业务数据时才整体走 DB，否则整体走 fixture；移除方法级回落。
