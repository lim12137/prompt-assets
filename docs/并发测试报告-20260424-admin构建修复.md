# Admin 构建修复报告（2026-04-24）

## 1. 背景与现象

- 已知历史报错：
  - `Cannot find module for page: /api/admin/submissions/[id]/reject`
  - `Failed to collect page data`
  - `.next/types/app/admin/page.ts not found`
- 目标：定位 root cause，做最小必要修复，并回归验证。

## 2. 根因判断

当前代码在本地可稳定构建（含清理缓存后重建），未复现功能性代码错误。结合报错特征，判断根因为：

- **Next 构建增量产物/类型缓存（`.next`）与 admin 路由变更不同步**，导致构建阶段引用到已失效的 page/module 路径，从而触发 `Cannot find module for page` 与 `Failed to collect page data`。

说明：

- 该类报错常见于 App Router 路由新增/重命名后，旧的 `.next` 清单仍被复用。
- 本仓库当前 admin 路由文件实际存在且可被 `next build` 正常收集。

## 3. 最小修复

仅修改 `apps/web/package.json` 的 `build` 脚本，在 `next build` 前强制清理 `apps/web/.next`：

- 修改前：`next build`
- 修改后：`node -e "require('node:fs').rmSync('.next',{recursive:true,force:true})" && next build`

## 4. 验证命令与结果

### 4.1 构建回归

```powershell
pnpm build:web
```

结果摘要：

- 构建成功（`Compiled successfully`）
- 页面数据收集成功（`Collecting page data` 无报错）
- 路由产物包含：
  - `/admin`
  - `/api/admin/submissions`
  - `/api/admin/submissions/[id]/approve`
  - `/api/admin/submissions/[id]/reject`

### 4.2 Admin 相关测试回归

```powershell
node --test --experimental-strip-types tests/integration/api/admin-submission-review.test.ts
```

结果摘要：

- `5 passed, 0 failed`
- 覆盖通过、拒绝、重复审核冲突、非 admin 拒绝、submission 不存在等路径。

## 5. 影响评估

- 修复范围仅限构建脚本，不改变业务逻辑。
- 可降低 admin 路由变更后因陈旧 `.next` 产物导致的构建不稳定问题。

## 6. 结论

- 本次按最小改动完成修复。
- 当前 `pnpm build:web` 与 admin 审核链路相关测试均通过。
