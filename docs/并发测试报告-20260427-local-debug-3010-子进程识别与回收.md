## 真机验证（Windows）
### 执行步骤（脚本化）
1. 后台启动旧链路：`pnpm dev:web`
2. 记录 3010 旧监听子 PID/父 PID
3. 后台执行：`pnpm local:web:restart`
4. 轮询校验：
   - 旧 PID 是否退出
   - 新 PID 是否接管 3010
   - `http://127.0.0.1:3010/api/health` 是否 200

### 实际结果
- 旧 PID：`7176`
- 旧父 PID：`28852`
- 新 PID：`6672`
- 新父 PID：`26740`
- 旧 PID 存活：`false`
- 旧父 PID 存活：`false`
- HTTP 状态：`200`
- HTTP 成功：`true`

### 命令行特征核对
- 旧/新监听子进程命令行均为：`...next\\dist\\server\\lib\\start-server.js`
- 旧/新父进程命令行均为：`...next\\dist\\bin\\next dev --hostname 127.0.0.1 --port 3010`

## 结论
- 已实现：这类“监听子进程 + next dev 父进程”被判定为本仓库可回收 Web 进程。
- 已验证：旧 PID 被自动结束，新 PID 成功接管 3010 且健康检查返回 200。
