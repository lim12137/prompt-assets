# 测试记录（2026-04-24）二期收口阻断修复：去 slug 外显与员工标识

## 背景阻断
1. Prompt slug 仅内部使用，前端不应要求输入或对用户外显。  
2. 员工候选提交不能硬编码 `alice@example.com`，需要最小可用的员工标识输入/切换。

## TDD Red

### 命令
```powershell
pnpm exec playwright test tests/e2e/admin/create-import-flow.spec.ts
pnpm exec node --test --experimental-strip-types tests/integration/api/prompt-create.test.ts tests/integration/api/admin-prompts-import.test.ts
```

### 失败摘要
- 管理页仍显示 `js-code-reviewer` / `landing-copy-framework` slug（应隐藏）。
- 创建页仍存在 `Slug` 输入（应去除）。
- 创建/导入 API 在未传 slug 时返回 400（应支持内部推导）。

## 最小实现
- `POST /api/prompts`：`slug` 改为可选，缺失时由标题内部推导。
- `POST /api/admin/prompts/import`：单项 `slug` 改为可选，缺失时由标题内部推导。
- 创建页：去掉 slug 输入与反馈外显，交互仍保持最小可用。
- 导入页：示例 JSON 去掉 slug，成功反馈改为标题列表。
- 管理页：去掉对用户展示的 prompt slug 字段。
- 详情页动作区：新增“当前员工邮箱”输入，点赞/候选提交均使用该值；候选提交不再硬编码 `alice@example.com`。
