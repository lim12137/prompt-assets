# 方案3 P0 冻结清单 / ADR 基线

文档版本：v1.0  
冻结日期：2026-04-23  
适用范围：方案3（极简提示词资产平台）一期实施前基线

## 1. 背景与目的

基于现有设计与技术审核结论，方案3已具备可推进条件，但在技术路线、部署拓扑、角色模型、审核对象、版本正文存储、API 命名等关键口径上存在冲突。若直接进入实现，将造成开发分叉、测试口径不一致与返工。

本文件作为实施前 P0 冻结基线，目标是：

- 统一阻断级决策，建立唯一实现口径。
- 将“审核意见”转为“可执行 ADR 决策”。
- 为后续代码实现、测试门禁与文档更新提供同一依据。

## 2. 本次冻结范围

本次仅冻结 P0 阻断项，覆盖以下六个主题：

- 技术路线基线。
- 部署拓扑基线。
- 角色模型基线。
- 审核对象语义基线。
- 版本正文存储策略基线。
- API 命名基线。

不涉及 UI 视觉细节、二期能力和非阻断性能优化。

## 3. P0 决策项（ADR）

## ADR-P0-001 技术路线基线（Open WebUI 改造 vs Next.js 独立实现）

当前冲突：`docs/04` 主张“基于 Open WebUI 改造”；`implementation plan` 主张“全新仓库 + Next.js 独立实现”。  
推荐决策：冻结为 **Next.js 独立实现**（单仓、前后端同仓，Route Handlers 承载 API）。  
理由：与现有实施计划、TDD 任务拆解和测试路径一致，且能避免 Open WebUI 定制带来的产品定位噪音与改造不确定性。  
影响范围：代码骨架、目录结构、测试脚本、CI 命令、部署镜像。  
待同步文档：`docs/04-技术方案与架构设计.md`、`docs/05-内网Docker部署方案.md`、`docs/06-实施计划-TDD.md`。

## ADR-P0-002 部署拓扑基线（是否 nginx/api 拆分）

当前冲突：`design` 文档存在 `nginx + web + api + postgres` 拓扑；`implementation plan` 为 `web + postgres` 极简形态。  
推荐决策：一期冻结为 **不拆分 nginx/api**，采用 **web + postgres** 的 Compose 最小拓扑。  
理由：降低首期复杂度，保证主链路先交付；与当前实施计划和验收命令一致。反向代理与独立 API 服务作为 P1 演进项。  
影响范围：Compose 文件、端口规划、健康检查、运维手册、验收脚本。  
待同步文档：`docs/plans/2026-04-23-方案3-极简提示词资产平台-design.md`、`docs/05-内网Docker部署方案.md`、`docs/07-测试与验收方案.md`。

## ADR-P0-003 角色模型基线（两角色还是三角色）

当前冲突：`docs/04` 为 `viewer/editor/admin` 三角色；`design` 为 `user/admin` 两角色。  
推荐决策：一期冻结为 **两角色模型：user/admin**。  
理由：一期核心流程仅需“提交方”和“审核方”职责分离；三角色会增加鉴权矩阵复杂度与测试面，当前收益不足。  
影响范围：用户表 `role` 枚举、接口鉴权、页面可见性、测试用例权限矩阵。  
待同步文档：`docs/04-技术方案与架构设计.md`、`docs/06-实施计划-TDD.md`、`docs/07-测试与验收方案.md`、`docs/plans/2026-04-23-方案3-极简提示词资产平台-implementation.md`。

## ADR-P0-004 审核对象基线（submission vs version）

当前冲突：部分描述以 `prompt_submission` 为审核对象，部分描述以 `prompt_version` 状态流转为审核对象，语义存在分裂。  
推荐决策：冻结为 **以 submission 为审核流程对象，以 version 为内容载体**。  
理由：审核队列和权限语义应聚焦“提交动作”；版本记录聚焦“内容快照与发布状态”，职责清晰，便于审计。  
影响范围：数据模型关联（`submission.candidate_version_id`）、审核 API 路径、审核列表查询、审计日志 target_type。  
待同步文档：`docs/04-技术方案与架构设计.md`、`docs/plans/2026-04-23-方案3-极简提示词资产平台-design.md`、`docs/reviews/2026-04-23-方案3-技术方案审核报告.md`。

## ADR-P0-005 版本正文存储策略基线（DB/文件/双写）

当前冲突：现有文档同时出现“DB 存正文”“文件快照存正文”“DB+文件双写”三种口径，且双写原子性与补偿机制未定义。  
推荐决策：一期冻结为 **DB 单写为唯一真源**；文件快照不进入 P0 主链路。  
理由：直接消除双写一致性风险与补偿复杂度；满足一期浏览、复制、投稿、审核、回滚主链路。文件快照可在 P1 以异步导出或归档任务补齐。  
影响范围：`prompt_versions` 字段设计、事务边界、备份恢复方案、并发一致性测试。  
待同步文档：`docs/04-技术方案与架构设计.md`、`docs/plans/2026-04-23-方案3-极简提示词资产平台-design.md`、`docs/reviews/2026-04-23-方案3-技术方案审核报告.md`、`docs/07-测试与验收方案.md`。

## ADR-P0-006 API 命名基线

当前冲突：存在 `admin/reviews/*` 与 `admin/submissions/*` 并存，及个别动作型命名不一致（如 `copy-log`）。  
推荐决策：冻结为 **资源导向命名**，统一使用：

- 前台：`/api/prompts`、`/api/prompts/{slug}`、`/api/prompts/{slug}/versions`、`/api/prompts/{slug}/submissions`、`/api/prompts/{slug}/like`
- 后台：`/api/admin/submissions`、`/api/admin/submissions/{id}/approve`、`/api/admin/submissions/{id}/reject`

理由：与审核对象基线一致，避免首版后二次重命名破坏兼容与测试资产。  
影响范围：接口实现、SDK/调用方、集成测试路径、验收脚本与文档示例。  
待同步文档：`docs/04-技术方案与架构设计.md`、`docs/plans/2026-04-23-方案3-极简提示词资产平台-design.md`、`docs/plans/2026-04-23-方案3-极简提示词资产平台-implementation.md`、`docs/07-测试与验收方案.md`。

## 5. 未纳入本次冻结的事项

以下议题不属于本次 P0 冻结，纳入后续里程碑评估：

- `nginx + 独立 api` 拆分拓扑及灰度发布策略。
- 文件快照或对象存储归档能力（含异步任务与对账）。
- 三角色模型（`viewer/editor/admin`）扩展及精细化权限矩阵。
- 审核流升级为多级审批、会签或策略路由。
- 搜索增强（全文检索、语义检索）与性能优化细节。
- 监控告警、可观测性体系与容量压测阈值深度化。
