## 背景
- 任务目标：首页去除“创建/导入暂未实现”占位，改为可用入口并直达：
  - `/admin/create`
  - `/admin/import`
- 约束：不回退既有未提交改动（尤其是 `tests/e2e/*`）。

## 代码改动
- 首页入口实现由按钮占位改为真实链接，保持现有按钮视觉风格。
- 新增首页入口配置常量，便于静态/单元验证。
- 新增单元测试验证入口文案与链接目标。
- 兼容更新视觉基线用例中的“创建”入口定位方式（按钮 -> 链接）。

### 4) 降级验证（按要求：单独起服务验证）
说明：
- 因 Playwright `webServer` 不稳，改为单独启动 `@prompt-management/web` 于 `127.0.0.1:3211`，并进行 HTTP 页面可用性校验。

验证点与结果：
- 首页 `/`
  - 存在链接：`/admin/import`、`/admin`、`/admin/create`
  - 不包含“暂未实现”
  - 包含新状态文案：“入口已开放，可直接进入创建、导入或管理页。”
- 创建页 `/admin/create`
  - 存在标题“创建提示词”
  - 存在到 `/admin/import` 的链接
- 导入页 `/admin/import`
  - 存在标题“批量导入提示词”
  - 存在到 `/admin/create` 的链接

## 结论
- 本次首页入口治理目标已实现：
  - 首页不再出现“创建/导入暂未实现”占位交互
  - 首页“创建/导入/管理”均为可直达入口，并与既有页面链路打通
- Playwright `webServer` 在当前环境存在不稳定（含 OOM），已采用“单独启动服务”的可行替代验证路径完成验收。
