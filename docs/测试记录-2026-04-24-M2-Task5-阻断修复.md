# 测试记录（2026-04-24）M2-Task5 阻断修复（investigate + TDD）

## 根因确认
- 详情页 404：`auto` 模式下仓储优先读 DB；当前环境 DB 可达但缺少目标 prompt，`getPromptDetail` 直接返回 `null`，未回落到 fixture。
- 聚合取错：候选卡“最新”排序在前端按 `submittedAt` 优先，而 fixture 中 `submittedAt` 与 `versionNo` 口径不一致，导致同一 submitter 可能选错。
- 候选提交失败：详情页已回落 fixture 后，提交 API 仍优先走 DB；DB 缺 prompt 时返回 `prompt not found`，导致 e2e 主路径失败。

## 修复策略
- `getPromptDetail`：`auto` 模式下 DB miss 回落 fixture，恢复详情主路径可用性。
- `createPromptSubmission`：`auto` 模式下 DB miss 回落 fixture，恢复候选提交主路径。
- 候选聚合排序：同一 submitter 取“最新”改为 `versionNo` 优先、`submittedAt` 兜底，保证 fixture/DB 口径更一致。
- 回归测试补强：详情 e2e 增加“同一 submitter 连续提交两次后仅展示最新候选卡正文”断言。
